## 1. 제품 개요

### 1.1 제품 비전

LUXSTAY는 고급 여행자와 프리미엄 숙박 업체를 연결하는 숙박 예약 플랫폼입니다. 복잡한 회원가입 없이도 아름다운 숙소 이미지와 직관적인 예약 플로우를 통해, 사용자가 몇 번의 클릭만으로 특별한 숙박 경험을 예약할 수 있도록 합니다.

### 1.2 배경 및 문제 정의

현재 숙박 예약 시장의 주요 문제점:

- 강제적인 회원가입으로 인한 높은 이탈률
- 복잡하고 불편한 예약 프로세스
- 숙소의 실제 분위기를 충분히 전달하지 못하는 빈약한 이미지
- 프리미엄 숙박에 특화된 큐레이션 서비스 부재

LUXSTAY는 이러한 문제를 해결하여 사용자 경험 중심의 프리미엄 숙박 예약 서비스를 제공합니다.

### 1.3 목표 사용자 (Target Users)

| 페르소나 | 특징 | 주요 니즈 |
|----------|------|-----------|
| 프리미엄 여행자 | 30-50대, 고소득층, 여행 빈도 높음 | 빠른 예약, 고퀄리티 이미지 확인 |
| 커플 / 허니무너 | 특별한 날 기념을 위한 예약 | 로맨틱한 숙소 분위기 확인 |
| 비즈니스 출장자 | 출장 중 프리미엄 숙박 선호 | 간편하고 빠른 예약 완료 |

## 2. 제품 범위 (Scope)

### 2.1 In-Scope (MVP 포함)

- 홈 화면 — 숙박 업체 리스트 및 추천 큐레이션
- 숙소 상세 페이지 — 고화질 이미지 갤러리 및 정보
- 예약 플로우 — 비회원 간편 예약
- 예약 확인 — 이메일/SMS 예약 확인서 발송

### 2.2 Out-of-Scope (MVP 제외)

- 회원가입 / 로그인 기능
- 마이페이지 / 예약 내역 조회
- 숙박 업체 파트너 관리 대시보드
- 리뷰 및 평점 시스템
- 결제 환불 / 취소 자동화

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 홈 화면 — 숙박 업체 리스트

#### 3.1.1 핵심 기능

- 숙박 업체 카드 리스트 표시 (그리드/리스트 전환 가능)
- 각 카드에 대표 이미지, 숙소명, 위치, 가격, 평점 표시
- 필터 기능: 위치, 가격대, 숙소 타입, 체크인-체크아웃 날짜
- 정렬 기능: 추천순, 가격 낮은순/높은순, 평점순
- 무한 스크롤 또는 페이지네이션

#### 3.1.2 UI 요구사항

- 카드 이미지는 최소 400x300px 이상 표시
- Hover 시 이미지 슬라이드 프리뷰 (2-3장)
- 즐겨찾기 버튼 (하트 아이콘) — 로컬 스토리지 저장
- '특가', '신규', '인기' 등 배지(Badge) 표시

#### 3.1.3 검색

- 상단 검색바: 도시/지역명 텍스트 검색
- 날짜 선택: 달력 팝업 (체크인/체크아웃)
- 인원 선택: 성인, 아동 인원 조절

---

### 3.2 숙소 상세 페이지 — 이미지 갤러리

#### 3.2.1 이미지 요구사항

- 메인 히어로 이미지: 최소 1200x800px, 풀스크린 지원
- 이미지 갤러리: 최소 10장 이상 등록 가능
- 갤러리 뷰: 그리드 섬네일 + 라이트박스 전체화면
- 좌우 스와이프 내비게이션 (모바일 지원)
- 이미지 카테고리: 외관, 객실, 욕실, 부대시설, 뷰

#### 3.2.2 숙소 정보

- 숙소명, 위치(지도 연동), 별점, 리뷰 수
- 주요 어메니티 아이콘 목록 (Wi-Fi, 주차, 수영장 등)
- 숙소 소개 텍스트 (접기/펼치기)
- 호스트 정보 및 연락처
- 위치 지도 (Google Maps 또는 Kakao Maps 임베드)

---

### 3.3 예약 플로우 — 간편 예약

#### 3.3.1 예약 단계 (3-Step)

1. **Step 1**: 날짜 및 인원 선택 — 가격 실시간 계산
2. **Step 2**: 예약자 정보 입력 — 이름, 연락처(휴대폰), 이메일
3. **Step 3**: 결제 정보 입력 → 예약 확정

#### 3.3.2 예약자 정보 입력

- **필수**: 예약자명, 휴대폰 번호, 이메일 주소
- **선택**: 특별 요청사항 (텍스트 입력)
- 비회원 예약 확인을 위한 예약번호 + 이메일 인증 방식

#### 3.3.3 결제

- 지원 결제수단: 신용카드/체크카드, 카카오페이, 네이버페이
- 결제 전 최종 금액 명세서 표시 (숙박 요금, 세금, 서비스 수수료)
- 결제 완료 시 즉시 예약 확인 이메일 발송

#### 3.3.4 예약 확인

- 예약 확인 페이지: 예약번호, 숙소명, 날짜, 금액
- 이메일 확인서: PDF 첨부 또는 모바일 바우처 형태
- SMS 문자 발송: 예약번호 + 체크인 안내

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 | 측정 기준 |
|------|----------|-----------|
| 성능 | 페이지 로딩 속도 | LCP 2.5초 이내 (Core Web Vitals) |
| 이미지 로딩 | 지연 로딩(Lazy Loading) 적용 | 초기 로딩 이미지 최대 5장 |
| 반응형 | 모바일/태블릿/데스크탑 지원 | 320px ~ 1920px 화면 대응 |
| 가용성 | 서비스 가동률 | 99.9% Uptime (월 다운타임 44분 이하) |
| 보안 | 결제 정보 암호화 | PCI DSS 준수, HTTPS 필수 |
| 접근성 | 웹 접근성 | WCAG 2.1 AA 수준 준수 |

## 5. 화면 흐름 (User Flow)

핵심 예약 사용자 흐름:

```
홈 화면 → 숙소 검색/필터 → 숙소 상세 → 날짜/인원 선택 → 예약자 정보 입력 → 결제 → 예약 확인
```

### 5.1 주요 화면 목록

| # | 화면명 | 주요 구성요소 |
|---|--------|---------------|
| 1 | 홈 / 숙소 목록 | 검색바, 필터, 숙소 카드 그리드, 추천 섹션 |
| 2 | 숙소 상세 | 이미지 갤러리, 숙소 정보, 예약 위젯(우측 고정), 지도 |
| 3 | 예약 Step 1 | 달력 날짜 선택, 인원 설정, 가격 요약 |
| 4 | 예약 Step 2 | 예약자 정보 폼, 특별 요청사항 |
| 5 | 예약 Step 3 | 최종 금액 명세, 결제 수단 선택, 결제 버튼 |
| 6 | 예약 완료 | 예약번호, 체크인/아웃 정보, 이메일 발송 안내 |

## 6. 디자인 방향성

### 6.1 브랜드 컨셉

- **키워드**: 럭셔리(Luxury), 신뢰(Trust), 편안함(Comfort)
- **톤앤무드**: 다크 골드 계열의 고급스러운 색상, 넓은 여백
- **이미지 퀄리티**: 프로페셔널 사진 촬영 필수, 최소 해상도 기준 명시

### 6.2 핵심 UI 원칙

- **이미지 퍼스트(Image First)**: 숙소의 아름다움이 가장 먼저 눈에 들어올 것
- **마찰 없는 예약(Frictionless Booking)**: 최소한의 입력으로 최대한 빠른 예약
- **신뢰 신호(Trust Signals)**: 명확한 가격 정책, 취소 정책 전면 노출

## 7. 성공 지표 (KPI)

| 지표 | 목표 (출시 3개월) | 측정 방법 |
|------|-------------------|-----------|
| 예약 전환율 | ≥ 3.5% | (예약 완료 수 / 상세 페이지 방문 수) × 100 |
| 예약 완료 소요 시간 | ≤ 5분 | Step 1 시작 ~ 예약 완료 페이지 도달 시간 |
| 이미지 로딩 만족도 | 이탈률 ≤ 30% | 갤러리 진입 후 3초 내 이탈 비율 |
| 예약 오류율 | ≤ 0.5% | 결제 실패 및 예약 오류 발생 비율 |
| 순추천지수 (NPS) | ≥ 50점 | 예약 완료 후 만족도 설문 (이메일 발송) |

## 9. 의존성 및 리스크

### 9.1 외부 의존성

- **결제 게이트웨이**: 토스페이먼츠 또는 아임포트 연동
- **지도 서비스**: Kakao Maps API 또는 Google Maps API
- **이메일 발송**: AWS SES 또는 SendGrid
- **SMS 발송**: 카카오 알림톡 또는 NHN Cloud SMS
- **이미지 CDN**: AWS CloudFront 또는 Cloudflare Images

### 9.2 리스크

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|-----------|
| 결제 연동 지연 | 높음 | 결제 게이트웨이 계약 및 테스트 계정 선개설 |
| 고화질 이미지 로딩 성능 저하 | 중간 | WebP 포맷 변환, CDN + Lazy Loading 적용 |
| 비회원 예약 오남용 (어뷰징) | 중간 | reCAPTCHA, 이메일 인증 필수화 |
| 초기 숙박 업체 파트너 확보 부족 | 높음 | 론칭 전 최소 20개 프리미엄 파트너 계약 목표 |

## 10. 용어 정리 (Glossary)

| 용어 | 정의 |
|------|------|
| 비회원 예약 | 계정 생성 없이 예약자 정보(이름, 연락처, 이메일)만으로 예약을 완료하는 방식 |
| 예약 바우처 | 예약 완료 후 발급되는 예약 확인서 (이메일 첨부 PDF 또는 모바일 화면) |
| 라이트박스 | 이미지 클릭 시 배경을 어둡게 하고 이미지를 전체 화면으로 표시하는 UI 패턴 |
| Trust Signal | 사용자의 신뢰를 높이는 UI 요소 (가격 보장 배지, 취소 정책 강조 등) |
| LCP | Largest Contentful Paint. 페이지 로딩 성능을 측정하는 Core Web Vitals 지표 |
